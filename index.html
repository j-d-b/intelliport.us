Dynamic human resource allocation based on demand
DSM
Dynamic Schedule Module

IntelliPort is developing the dynamic schedule module to automate and optimize creation of staff schedules to minimize cost of human resources.

The DSM offers two unique tools:

  • A simulator
  • A schedule generator

The simulator helps recruitment planners determine hiring needs by generating the number of required new exployees using a rolling basis demand over a selected time interval (e.g. weeks, months, years).

The schedule generator builds cost-efficient schedules based on forecasted demand. It remains flexible by allowing schedules to be updated as demand changes.

The DSM can integrate with a Terminal Operating System (TOS) to dynamically receive the demand for work. It can also integrate with an Enterprise Resource Planning (ERP) system to provide working hours for each employee, including overtime and on-call hours, and thus allow for immediate payroll generation.

Minimizing unproductive container moves
CRPM
Container Relocation Problem Module

IntelliPort is beginning development of a smart module that uses expected container retrieval order to maximize yard efficiency by minimizing the number of future relocations that are caused by the placement of blocking or new containers.

The CRPM has two functions:

  1. Determining placement within a block for relocation of a blocking container during retrieval of a requested container
  2. Determining placement within a block when stacking a new container

At a high level, the CRP software takes in a container ID to retrieve (1) or a block location to stack a new container (2) and outputs the optimized set of moves that should be taken to complete this action.

CRPM receives data from a Terminal Operating System (TOS) about containers currently stacked in the yard to determine departure order and sends the list of optimal moves to the TOS as instructions for use by crane operators. The module will continuously refresh its information and update container expected retrieval order in real-time based on changes in the yard.

If a terminal has implemented a Prediction System (PS) for container retrieval times or Truck Appointment System (TAS), CRPM will also receive data from these.
